Add a permanent Android APK download route to the No Read server.

Goal:
The site page download.html downloads the app from:
GET /app/noread.apk

Do not rewrite the server. Add a simple safe static APK download endpoint.

Requirements:
1. Put the APK file on the server at:
   data/app/noread.apk
   or
   downloads/noread.apk

2. Add public route:
   GET /app/noread.apk

3. It must return the APK with headers:
   Content-Type: application/vnd.android.package-archive
   Content-Disposition: attachment; filename="NoRead.apk"
   Cache-Control: no-store

4. If the file does not exist, return JSON:
   { "success": false, "message": "No Read Android app is not uploaded yet." }

5. Keep CORS working for:
   https://noread.rubby-studios.com
   https://api.rubby-studios.com

6. Do not require login for this route.

Example Express route:
const path = require('path');
const fs = require('fs');

router.get('/noread.apk', (req, res) => {
  const apkPath = path.join(process.cwd(), 'data', 'app', 'noread.apk');
  if (!fs.existsSync(apkPath)) {
    return res.status(404).json({ success: false, message: 'No Read Android app is not uploaded yet.' });
  }
  res.setHeader('Content-Type', 'application/vnd.android.package-archive');
  res.setHeader('Content-Disposition', 'attachment; filename="NoRead.apk"');
  res.setHeader('Cache-Control', 'no-store');
  return res.sendFile(apkPath);
});

Mount it as:
app.use('/app', appDownloadRoutes);
